# Montessori Kindergarten Baobab — Website

This is the website for **Baobab e.V.**, a bilingual (German/English) Montessori kindergarten in central Munich, founded as a parents' initiative in 1996.

🌐 **Live site:** [kindergarten-baobab.de](https://kindergarten-baobab.de)

----

## About this repository

This repository contains the source files for the Baobab kindergarten website. It is connected to [Netlify](https://netlify.com), which automatically publishes any changes within about 30 seconds of saving.

### Files

| File | Purpose |
|------|---------|
| `index.html` | The entire website — all content, design, and layout in one file |
| `sitemap.xml` | Tells Google which pages exist on the site |
| `robots.txt` | Tells search engines how to crawl the site |
| `README.md` | This guide |

---

## How to update the website

You do not need any coding experience to make updates. Follow these steps:

### 1. Describe your change to Claude
Go to [claude.ai](https://claude.ai) and describe what you want to change in plain English. For example:
- *"Update Sabine's job title to Senior Teacher"*
- *"Add a new testimonial from Maria, parent of two children"*
- *"Change the Tuesday closing time to 16:45"*
- *"Add a note about the summer closure dates"*

Paste the current `index.html` content into Claude if asked, and Claude will give you the updated file.

### 2. Edit the file in GitHub
1. Open this repository on [github.com](https://github.com)
2. Click on `index.html`
3. Click the **pencil icon ✏️** (Edit this file) in the top right
4. Select all the existing text and replace it with the new content from Claude
5. Scroll down to the **Commit changes** box
6. Write a short description of what you changed, for example: *"Updated summer closure dates 2025"*
7. Click **Commit changes**

### 3. Wait 30 seconds
Netlify will automatically detect the change and redeploy the live website. That's it.

---

## Good to know

- **Every save is recorded.** You can always go back to an older version by clicking **History** at the top of the file.
- **Never delete this repository.** All other mistakes can be undone — deletion cannot.
- **Keep a backup.** Download a copy of `index.html` to Google Drive every few months, or after major updates.
- **Don't worry about breaking things.** If something looks wrong after an update, the previous version is always one click away in the file history.
